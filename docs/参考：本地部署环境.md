# MUSA SDK Host Installation

This skill installs the full MUSA SDK stack on a bare-metal host. It expects all version tags to be provided upfront (by the user or a parent workflow) and proceeds directly to download and install.

## Input Variables

All version tags must be collected before invoking this skill. Use the question tool to gather any missing values.

| Variable | Example | Description |
|----------|---------|-------------|
| `MUSA_SDK_VERSION` | `4.3.5` | MUSA SDK version (used for toolkits and download path) |
| `MT_GPU_DRIVER_VERSION` | `3.3.5` | GPU driver version |
| `MT_MUDNN_VERSION` | `3.1.5` | muDNN version |
| `MT_MCCL_VERSION` | `2.1.5` | MCCL version |
| `MT_TRITON_VERSION` | `1.4.5` | Triton MUSA SDK tag (filename component, not the pip version) |
| `MT_MTML_VERSION` | `2.3.2` | MTML version |
| `MT_CONTAINER_TOOLKIT_VERSION` | `2.1.0` | MT Container Toolkit version |
| `MT_GPU_TYPE` | `S5000` | GPU type: S5000 or S4000 |
| `MT_GPU_ARCH` | `PH1` | Architecture suffix: PH1 (S5000) or QY2 (S4000) |
| `MT_PYTHON_VERSION` | `py311` | Python version for Triton: py310 or py311 |
| `TORCH_MUSA_VERSION` | `v2.7.1` | torch_musa release tag (for Docker image) |
| `TORCH_MUSA_DOCKER_IMAGE` | `4.3.5_kuae2.1_20260119_torch2.7.1_ubuntu` | Docker image tag |

## Prerequisites

Check and install required tools at the very start.

| Tool | Purpose | Install |
|------|---------|---------|
| `mc` | MinIO Client for MOSS downloads | `curl -O https://dl.min.io/client/mc/release/linux-amd64/mc && chmod +x mc && sudo mv mc /usr/local/bin/` |
| `jq` | JSON processor for state files | `sudo apt install -y jq` |

If a tool is missing, ask the user whether to install it automatically.

## Sudo Password

Check `MY_SUDO_PASSWD` env var first. If unset or invalid, prompt the user.

```bash
if [ -n "$MY_SUDO_PASSWD" ] && echo "$MY_SUDO_PASSWD" | sudo -S -v 2>/dev/null; then
    SUDO_PASSWORD="$MY_SUDO_PASSWD"
else
    # Use the question tool to prompt user
fi
```

Only use `SUDO_PASSWORD` for: installing system packages (jq, mc, linux-headers, dkms), driver deb, toolkits install.sh, muDNN install, MCCL install, MTML deb, Container Toolkit deb, Docker restart. Never for git or docker pull.

## State Persistence

Save state to `./.musa_sdk_install_state.json` before reboot and load on resume. Use `jq` to read fields.

### State Values

`initialized` | `downloaded` | `driver_installed` | `toolkits_installed` | `mudnn_installed` | `mtml_installed` | `mccl_installed` | `triton_installed` | `completed`

### Resume Logic

On start, if state file exists, ask user: **Resume** or **Start fresh**. On resume, load all variables from the file, verify driver with `mthreads-gmi`, then jump to the next step based on `INSTALL_STATE`.

### Save / Load

```bash
# Save (example)
cat > ./.musa_sdk_install_state.json << EOF
{ "MUSA_SDK_VERSION":"${MUSA_SDK_VERSION}", "MT_GPU_DRIVER_VERSION":"${MT_GPU_DRIVER_VERSION}",
  "MT_MUDNN_VERSION":"${MT_MUDNN_VERSION}", "MT_MCCL_VERSION":"${MT_MCCL_VERSION}",
  "MT_TRITON_VERSION":"${MT_TRITON_VERSION}", "MT_MTML_VERSION":"${MT_MTML_VERSION}",
  "MT_CONTAINER_TOOLKIT_VERSION":"${MT_CONTAINER_TOOLKIT_VERSION}",
  "MT_GPU_TYPE":"${MT_GPU_TYPE}", "MT_GPU_ARCH":"${MT_GPU_ARCH}",
  "MT_PYTHON_VERSION":"${MT_PYTHON_VERSION}",
  "TORCH_MUSA_VERSION":"${TORCH_MUSA_VERSION}",
  "TORCH_MUSA_DOCKER_IMAGE":"${TORCH_MUSA_DOCKER_IMAGE}",
  "INSTALL_STATE":"${state}", "WORK_DIR":"$(pwd)", "TIMESTAMP":"$(date -Iseconds)" }
EOF

# Load
MUSA_SDK_VERSION=$(jq -r '.MUSA_SDK_VERSION' ./.musa_sdk_install_state.json)
# ... repeat for each variable
```

## Installation Steps

Installation order: **Driver -> Toolkits -> muDNN -> MTML -> MCCL -> Triton -> Container Toolkit (optional) -> Docker Image**

### Step 1: Download Packages

```bash
mkdir -p ./musa_sdk_packages
mc alias set sh-moss https://sh-moss.mthreads.com sw-guest-mt-sw sw-guest123
BASE="sh-moss/sw-release/musa/external/${MUSA_SDK_VERSION}/deb"

mc cp "${BASE}/musa_${MT_GPU_DRIVER_VERSION}-server_amd64.deb" ./musa_sdk_packages/
mc cp "${BASE}/musa_toolkits_rc${MUSA_SDK_VERSION}.tar.gz" ./musa_sdk_packages/
mc cp "${BASE}/mudnn_rc${MT_MUDNN_VERSION}.${MT_GPU_ARCH}.tar.gz" ./musa_sdk_packages/
mc cp "${BASE}/mccl_rc${MT_MCCL_VERSION}.${MT_GPU_ARCH}.tar.gz" ./musa_sdk_packages/
mc cp "${BASE}/triton310_rc${MT_TRITON_VERSION}_${MT_PYTHON_VERSION}.tar.gz" ./musa_sdk_packages/
```

For MTML, search MOSS paths in order:
1. `sh-moss/sw-release/musa/external/${MUSA_SDK_VERSION}/others/`
2. `sh-moss/sw-release/musa/external/${MUSA_SDK_VERSION}/deb/`

Match pattern: `*mtml*${MT_MTML_VERSION}*amd64.deb`

For MT Container Toolkit, search:
1. `sh-moss/sw-build/cloud/runtime/ubuntu20.04/2.x/linux-amd64/`

Match: `*mt-container-toolkit*${MT_CONTAINER_TOOLKIT_VERSION}*amd64.deb`

Skip any package that already exists locally in `./musa_sdk_packages/`. Save state `downloaded`.

### Step 2: Install Driver

```bash
# Prerequisites
echo "$SUDO_PASSWORD" | sudo -S apt install -y linux-headers-$(uname -r) dkms

# Uninstall old driver if present
dpkg -s musa &>/dev/null && echo "$SUDO_PASSWORD" | sudo -S dpkg -P musa && sudo modprobe -rv mtgpu 2>/dev/null || true

# Install
echo "$SUDO_PASSWORD" | sudo -S apt install ./musa_sdk_packages/musa_${MT_GPU_DRIVER_VERSION}-server_amd64.deb

# Load driver without reboot when possible
echo "$SUDO_PASSWORD" | sudo -S modprobe mtgpu
```

Save state `driver_installed`. **Never run `sudo reboot` automatically.** Prefer manual driver reload first. Only ask the user to reboot manually if `modprobe` loading does not make the driver available.

Verify with `mthreads-gmi`. If it succeeds, continue without reboot. If it still fails because the new driver is not fully loaded, then ask the user to reboot manually and resume.

### Step 3: Install MUSA Toolkits

```bash
cd ./musa_sdk_packages && tar -zxf musa_toolkits_rc${MUSA_SDK_VERSION}.tar.gz && cd musa_toolkits_install
echo "$SUDO_PASSWORD" | sudo -S bash ./install.sh -u   # uninstall old
echo "$SUDO_PASSWORD" | sudo -S bash ./install.sh      # install new
```

Add to `~/.bashrc` and `~/.zshrc` (if not already present):

```bash
export MUSA_HOME=/usr/local/musa
export PATH=$MUSA_HOME/bin:$PATH
export LD_LIBRARY_PATH=$MUSA_HOME/lib:$LD_LIBRARY_PATH
```

Verify: `musa_version_query` and `musaInfo`. Save state `toolkits_installed`.

### Step 4: Install muDNN

```bash
cd ./musa_sdk_packages && tar -zxf mudnn_rc${MT_MUDNN_VERSION}.${MT_GPU_ARCH}.tar.gz && cd mudnn
echo "$SUDO_PASSWORD" | sudo -S ./install_mudnn.sh -i
```

Verify: `ls /usr/local/musa/lib/libmudnn.so*`. Save state `mudnn_installed`.

### Step 5: Install MTML

```bash
echo "$SUDO_PASSWORD" | sudo -S dpkg -i ./musa_sdk_packages/*mtml*amd64.deb
```

Verify: `dpkg -s mtml` and `mthreads-gmi`. Save state `mtml_installed`.

### Step 6: Install MCCL

Prerequisite: MTML must be installed (libmtml.so required).

```bash
cd ./musa_sdk_packages && tar -zxf mccl_rc${MT_MCCL_VERSION}.${MT_GPU_ARCH}.tar.gz && cd mccl
echo "$SUDO_PASSWORD" | sudo -S bash ./install.sh
```

Verify: `/usr/local/musa/bin/mccl_version`. Save state `mccl_installed`.

### Step 7: Install Triton

The required Python (`python3.10` or `python3.11` per `MT_PYTHON_VERSION`) must be available. If missing, **never fall back to a different Python version**. Ask the user for a conda/venv path or install the correct Python via deadsnakes PPA.

```bash
cd ./musa_sdk_packages && tar -zxf triton310_rc${MT_TRITON_VERSION}_${MT_PYTHON_VERSION}.tar.gz
python3.XX -m pip install triton-*.whl   # use the matching python
```

Verify: `python3.XX -c "import triton; print(triton.__version__)"`. Save state `triton_installed`.

### Step 8: Install MT Container Toolkit (Optional)

Ask the user whether they need container GPU pass-through. Skip if not needed.

```bash
echo "$SUDO_PASSWORD" | sudo -S DEBIAN_FRONTEND=noninteractive dpkg -i ./musa_sdk_packages/*mt-container-toolkit*amd64.deb
echo "$SUDO_PASSWORD" | sudo -S /usr/bin/musa/docker setup /usr/bin/musa
echo "$SUDO_PASSWORD" | sudo -S systemctl restart docker
```

Verify: `dpkg -s mt-container-toolkit` and test `docker run --rm --gpus all <image> mthreads-gmi`. Save state `completed`.

### Step 9: Pull Docker Image

Check if image exists locally first. If not:

```bash
# Login (use DOCKER_SH_HARBOR_USER / DOCKER_SH_HARBOR_PASSWD env vars, or prompt)
echo "$DOCKER_SH_HARBOR_PASSWD" | docker login sh-harbor.mthreads.com -u "$DOCKER_SH_HARBOR_USER" --password-stdin
docker pull sh-harbor.mthreads.com/mcctest/musa-train:${TORCH_MUSA_DOCKER_IMAGE}
```

Do NOT use SUDO_PASSWORD for docker operations.

### Final Summary

Print a table of all installed components and their verification commands. Delete state file on success (or offer to keep it).

## Important Rules

1. **Never fall back Python version** -- if py311 is requested and python3.11 is missing, install it; never substitute py310.
2. **Never auto-reboot** -- prefer manual driver reload first; only tell the user to reboot manually if reload does not work.
3. **SUDO_PASSWORD scope** -- only for system package install, driver, toolkits, muDNN, MCCL, MTML, Container Toolkit, Docker restart. Never for git or docker pull.
4. **Skip existing downloads** -- check `./musa_sdk_packages/` before each `mc cp`.
5. **BIOS notes** -- S5000: disable IOMMU (VT-d). S4000 with >=256GB RAM: enable IOMMU.
