# 单机环境部署

# 驱动安装

:::
安装参考：[安装指导 | 摩尔线程文档中心](https://docs.mthreads.com/musa-sdk/musa-sdk-doc-online/install_guide)

驱动下载链接：[摩尔线程开发者 | Moore Threads](https://developer.mthreads.com/sdk/download/musa?equipment=&os=&driverVersion=&version=)
:::

*   安装依赖
    

```shell
sudo apt install lightdm
# 安装时，default display manager 选择 lightdm
sudo apt install dkms
sudo apt install libgbm1 libglapi-mesa
```

*   卸载驱动
    

```shell
# 查看驱动版本
dpkg -s musa

# 如果驱动版本不对：
# 卸载驱动
sudo dpkg -P musa

# 查询是否仍有 mtgpu ko 残留
lsmod | grep mtgpu
```

*   安装驱动
    

```shell
# 安装对应驱动包
sudo dpkg -i Path/To/xxxx.deb
# 优先手动重新加载驱动
modprobe -rv mtgpu && modprobe mtgpu
# 如果手动加载后驱动仍未生效，再手动重启
sudo reboot
```

*   验证
    

```shell
mthreads-gmi
```

# 安装Container Toolkit

:::
默认当前机器已安装 Docker

安装参考：[安装指导 | 摩尔线程文档中心](https://docs.mthreads.com/cloud-native/cloud-native-doc-online/install_guide)

软件栈下载链接：[摩尔线程开发者 | Moore Threads](https://developer.mthreads.com/sdk/download/CloudNative?equipment=&os=&driverVersion=&version=)
:::
```shell
# 安装工具
# 部分低版本需要在安装mt-container-toolkit之前额外安装 sgpu-dkms
sudo dpkg -i 6cf87f0b1_mtml_2.1.0-linux-R_amd64.deb
sudo dpkg -i mt-container-toolkit_2.1.0-1_amd64.deb

# 绑定
(cd /usr/bin/musa && sudo ./docker setup $PWD)

# 验证(可以正常输出 mthreads-gmi 内容)
docker run --rm --env MTHREADS_VISIBLE_DEVICES=all registry.mthreads.com/cloud-mirror/ubuntu:20.04 mthreads-gmi
```

# 构建容器环境

*   起容器
    

```shell
image_name=<image_name>
docker run -itd \
  --name=torch_musa_test \
  --env MTHREADS_VISIBLE_DEVICES=all \
  --shm-size=80g \
  --network=host \
  --privileged \
  --pid=host \
  -v /data:/data \
  $image_name bash
```

*   验证容器环境
    

```shell
# 1. 验证 MUSA（初步验证） 
musaInfo

# 2. 验证 pytorch 
python -c "import torch; print(torch.musa.is_available()); print(torch.tensor(1, device='musa') + 1);"
```
