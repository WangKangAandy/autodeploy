# 文档更新判定规则

## 无需更新文档的情况

以下变更通常不需要更新文档：

- 仅重命名局部变量或内部函数
- 仅调整内部实现，且满足：
  - 外部行为不变
  - 输入输出不变
  - 配置不变
  - 日志格式不变
  - 状态文件字段不变
- 仅修改测试文件
- 仅修改注释或示例（不改变行为）

## 必须更新文档的情况

以下变更必须更新文档：

| 变更类型 | 需更新的文档 |
|----------|-------------|
| 接口/函数签名变化 | 对应模块文档 |
| 配置项增删改 | 对应模块文档 |
| 状态文件字段变化 | state-manager.md |
| 调度/路由/trace 行为变化 | dispatcher.md 或 tracing.md |
| Skill 触发词、参数、能力变化 | skills.md |
| Intent 增删改 | skills.md |

## 判断口诀

```
外部可见 → 必须更新
内部重构 → 通常不更新
拿不准 → 先写理由，让 reviewer 判断
```

## Gate Verdict Rules

### PASS

满足以下任一条件：

- 未命中任何 DOC-MAP 规则
- 命中规则且相关文档已正确更新
- 命中规则但依据本文件可明确判定无需更新，且理由充分成立

### WARN

满足以下任一条件：

- 命中 medium/low severity 规则，文档未更新，但影响较小
- 作者给出了"无需更新"理由，但存在合理不确定性
- subagent 无法确定是否应该更新文档

### BLOCK

**默认 BLOCK 条件（三者同时满足）：**

1. high severity
2. 且发生外部可见行为 / 配置 / 状态 / trace 变化
3. 且相关文档未更新或"不更新理由"不成立

**也可 BLOCK 的异常情况：**

- 命中高风险规则但缺少必要说明
- 文档路径明显错误或缺失
- 更新文档与命中规则不匹配
- 配置/状态字段变化非常明确，但作者写了明显不成立的 N/A
